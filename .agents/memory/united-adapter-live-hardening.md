name: United portal adapter ‚Äî live-submit hardening & create-POST instrumentation
description: How United's multi-step ASP.NET/Salesforce create differs from SIT, and the redaction rule for capturing its create contract
---

United (partner.unitededucation.com/Manage/newapplication) is a Salesforce-backed **ASP.NET MVC + KT Stepper multi-step wizard**, NOT a single webhook like SIT ‚Äî so it cannot be collapsed to one HTTP call. The adapter drives the UI and, on the first LIVE run, instruments `page.on("requestfinished")` to capture the real create contract for a future Phase-2 HTTP replay.

**Redaction rule (critical):** capturing the create POST body must reveal the CONTRACT (every field NAME + structural Salesforce ids: contactid/programid/appid + the cascade select ids) but NEVER applicant PII values. Rule = **default-deny on values**: log all keys, redact every value except a narrow structural allowlist (`*_id`/`id$`, `select{university,program,degree,lang,campus}`, regtype/country/destination/degree/program/lang/campus, `university*`), and always redact keys matching a PII regex (name|passport|kimlik|phone|mail|address|birth|dob|dateinput|school|father|mother|national|tckn). Handle form-urlencoded (the real create) AND JSON; unknown format ‚Üí suppress entirely. Anti-forgery `__RequestVerificationToken` always redacted. Also strip URL query strings before logging (`url.split("?")[0]`) so `/Account/searchapp?word=<email>` doesn't leak. This mirrors SIT's "never log the request body (PII)" convention while still exposing the field names we need.

**Dry boundary:** `dryRun = doSubmit===false || PORTAL_DRYRUN==="1"`. Dry walks the KT-Stepper steps (term‚Üídegree‚Üíprogram cascade) and STOPS at the Program‚ÜíPersonal boundary ‚Äî before personal-info fills and before the final submit. The Salesforce application record is created ONLY at the final submit, so no student is created in dry. This existing behavior is spec-locked ("Deƒüi≈ütirme") ‚Äî do NOT add an aborting guard on step-transition POSTs; KT-Stepper "Continue" may POST to /Manage/newapplication per step and a guard would false-trip.

**Member gate:** submit() hard-skips any `profile.universityName` not in UNITED_ALLOWLIST (Biruni / Ni≈üanta≈üƒ± / Ankara Bilim ‚Äî exactly 3, folded fuzzy) ‚Üí `{skippedNotMember:true, routeTo:"direct"}`, same philosophy as SIT membership. Dedup via `GET /Account/searchapp?word=email|name` ‚Üí parse `{totalSize,records}`, log count+Id only, seed `#contactid` when found.

**Program step reuses `matchProgram`** against the LIVE select2 options (panel nameMap/general + synonyms), captures `availablePrograms` + `resolution:"not_in_dropdown"` on miss. select2 is driven programmatically: set `el.value` + dispatch `change` + `jQuery(el).trigger("change")`, then wait for cascade AJAX.

**Program cascade matching (fixes `programMissing` false-positives):** each cascade `<select>` (university‚Üíprogram‚Üídegree‚Üílang‚Üícampus) goes through `selById`, which WAITS for the AJAX-populated options (`waitForFunction`, 12s), LOGS the option texts, and matches flexibly via `looseMatchIndex` = `fold(strip "(...)")` then (a) normalized substring either-direction (reverse only len>4), (b) token-overlap ‚â•0.6. United labels carry parenthetical qualifiers the CRM name omits (e.g. `"Computer Engineering (Non-Thesis) (Turkish)"`) so plain `.includes()` never matched. Program step keeps `matchProgram` (panel-aware, thesis-aware) as the FIRST pass, then `looseMatchIndex` over live option TEXT as fallback ‚Äî stripping parens can pick a Thesis/Non-Thesis sibling, accepted only because matchProgram runs first. **Critical selects** (`CRITICAL_SELECT_IDS` = selectuniversity/program/degree/lang) NEVER auto-pick the first option on a miss (would submit the wrong value) ‚Äî only a LONE real option is forced. **Empty-target respect-preselection:** when a select is called with no target (selectlang/selectcampus), first honor an existing real `selectedIndex` (United derives degree/lang/campus from the program and auto-selects them) ‚Äî this stops critical `selectlang` from stalling the Program‚ÜíPersonal step. Selection always fires `selectedIndex`+`change`+`jQuery.trigger("change")` then waits 1200ms for the next cascade.

**Member gate EN‚ÜîTR matching (`isUnitedMember`):** apply sends English names ("Istanbul Nisantasi University") but UNITED_ALLOWLIST is Turkish ("Ni≈üanta≈üƒ± √úniversitesi"); plain `fold`+substring never matched ‚Üí wrongly routed to direct/SKIP (exclusive_region). Fix = `strip()` (fold + remove institution/city stopwords: university|universitesi|universite|univ|istanbul|the|of) then match by (a) name-contains-entry substring OR (b) token-subset (member iff ALL distinctive entry tokens appear in the name). CRITICAL: use ONLY `f.includes(e)` (name‚äáentry), NOT the reverse `e.includes(f)` ‚Äî the reverse false-positives non-member "Ankara University" (strips to "ankara") against entry "ankara bilim" (a real major non-member public uni). It's a strict 3-school gate so false-positives are worse than false-negatives; pure-translation variants like "Ankara Science University" (science‚âÝbilim) intentionally won't match ‚Äî add a curated alias only if seen in CRM data. Verify with the 10-case matrix (node --experimental-strip-types importing real fold).

**United portal redesigned ‚Üí 6-step KT-Stepper wizard (2026-07-06):** the old single-page `#selectuniversity/#selectprogram/#selectdegree` cascade model was WRONG ‚Äî it never advanced past screen 1, so every cascade select stayed empty ‚Üí programMissing:true. Real flow (submit() drives it between goto /Manage/newapplication and the atPersonal/dryRun boundary): Step1 term radio `input[name="radio_buttons_2"]` (check+click+change, global `clickradio()`) ‚Üí Continue; Step2 degree CARD radio (value=Bachelor/Master/PhD/Vocational School, click + global `clickradio1()`) ‚Üí Continue; Step3a `#selectuniversity` select2 filter set to member uni (option values are ENGLISH accent-free e.g. "Nisantasi University") ‚Üí wait grid AJAX; Step3b program is a CARD grid `div.single-table` (NOT a dropdown) ‚Äî fuzzy-match title, click `span.plan-submit` (jQuery delegation, needs real .click()) ‚Üí Continue. Registration Type `#regtype` MOVED to the Personal step (Step4) ‚Äî it's no longer on the first page. Hidden Salesforce linkage `#contactid/#accountid/#appid` auto-fill as the wizard progresses ‚Äî never seed them. Program-card match SAFETY (critical for live submits): accept only exact/near-exact normalized title containment (score 1) OR multi-token target with overlap ‚â•0.67 (single shared token must NOT pick a sibling like "Civil Engineering"); THEN hard-gate on the "Selected Majors (N)" counter incrementing after the click (fail-closed = programMissing stays true; a false-negative is safe, a false-positive submits the wrong program). Counter check is English-regex-coupled; if UI localizes, add a selected-CSS/disabled-button fallback signal.

**`__name is not defined` in page.evaluate:** esbuild/tsx wraps NAMED inner functions passed to page.evaluate with `__name(fn,"‚Ä¶")`, but `__name` doesn't exist in the browser context ‚Üí any evaluate containing a named inner arrow (e.g. `const nrm=(s)=>...`) throws (Step-1 evaluate had no named inner fn so it worked). Fix = ensureNameShim() that does `page.evaluate(()=>{ globalThis.__name ||= (f)=>f; })`, called right after goto AND at the end of clickContinue (KT-Stepper swaps steps client-side w/o reload, but re-assert per step in case a remount clears globals).

**Step-3 program grid: wait for cards + preserve Thesis tokens.** Two headless-only bugs surfaced live: (1) fixed `waitForTimeout(2500)` read the grid before the ~150 cards rendered (AJAX, slower headless) ‚Üí 0 cards, matched:false. Fix = `waitForFunction` (15s) gating on `div.single-table` present AND at least one card containing the university's distinctive core token (uniCore = longest token of folded universityName minus university/istanbul/etc.), then 1s settle; `.catch` fail-safe. (2) the program-match `nrm` was deleting parenthetical CONTENT (`replace(/\([^)]*\)/g," ")`), collapsing "(Thesis)" and "(Non-Thesis)" to the same string ‚Üí wrong variant. Fix = drop only punctuation‚Üíspace (keep tokens non/thesis/turkish) so the correct variant out-scores its sibling via token overlap. Card title shape: "<Uni> <Program> (<Lang>) (<Thesis|Non-Thesis>) <Degree>".
