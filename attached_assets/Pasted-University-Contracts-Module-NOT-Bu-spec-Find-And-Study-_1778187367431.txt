University Contracts Module
NOT: Bu spec, Find-And-Study-OS (EdCons OS) projesine eklenecek modülü tarif eder. Find-And-Study-OS başka bir Replit projesidir; aşağıdaki dosya yolları o projenin monorepo yapısına göredir (lib/db/src/schema/, artifacts/api-server/, artifacts/edcons/). Mevcut altyapı kullanılacak: Drizzle + PostgreSQL + Replit Object Storage + nodemailer + emailQueueTable + notificationsTable + notificationDispatcher.ts + mevcut contractChecker.ts (acente sözleşme expiry kontrolcüsünün birebir paterni).

What & Why
Üniversitelerle yapılan ortaklık sözleşmelerini admin panelden yönetmek. Her sözleşme:

Bir üniversiteye bağlı (universitiesTable.id)
Bir destinasyon/ülke ile filtrelenebilir
Yıl, başlangıç/bitiş tarihi, dosya (PDF) içerir
Statüsü otomatik hesaplanır: active / expiring_soon (30 gün içinde bitecek) / expired / no_dates
Süresi dolmadan önce e-posta + uygulama içi bildirim gönderir (30/14/7/1 gün kala + bittiği gün)
Done looks like
Admin "University Contracts" sayfasına girer; destinasyona, yıla, üniversiteye, statüye göre filtreleyebilir
"Add Contract" butonu → form: üniversite (autocomplete from universitiesTable), destinasyon (autocomplete from destinationsTable), yıl, başlangıç tarihi, bitiş tarihi, sözleşme dosyası (PDF/DOCX upload), notlar
Listede her sözleşme satırında: üniversite logosu + adı, destinasyon (bayrak emoji + ülke), yıl, tarih aralığı, kalan gün sayısı, statü badge'i (renkli), download + edit + delete
Hover'da kalan gün ya da geçmiş gün sayısı tooltip'te görünür
Süresi dolmaya 30/14/7/1 gün kala, bitiş gününde ve sonrasında otomatik e-posta uyarısı admin'e ve atanmış staff'a gönderilir (her uyarı bir kez)
Notification merkezinde de aynı uyarı gözükür (notificationsTable)
Permissions: yeni university_contracts kategorisi (view/manage)
Out of scope
Üniversite tarafının imzalaması (DocuSign vb.) — sadece dosya yükleme/yönetimi
Sözleşme şablonu üretme — bu modül "yüklenen sözleşmeleri yönetir", "üretmez" (üretim Task #1'de)
Çoklu dil sözleşme paketi — tek bir dosya alanı
Steps
1. Schema (lib/db/src/schema/universityContracts.ts)
universityContractsTable:
  id (uuid, pk)
  universityId (uuid, fk → universities.id, not null)
  destinationId (uuid, fk → destinations.id, nullable — opsiyonel; üniversitenin country alanından da çıkarılabilir)
  country (text, not null) — denormalize: filter performansı için
  year (int, not null)
  effectiveDate (date, nullable)
  expiryDate (date, nullable)
  fileObjectKey (text, not null) — Object Storage path
  fileName (text, not null) — orijinal dosya adı
  fileMimeType (text, not null)
  fileSizeBytes (int)
  notes (text, nullable)
  status (computed via SQL — runtime'da hesapla, kolon olarak tutmaya gerek yok; ama "lastNotifiedX" alanları lazım)
  lastWarning30SentAt (timestamp, nullable)
  lastWarning14SentAt (timestamp, nullable)
  lastWarning7SentAt (timestamp, nullable)
  lastWarning1SentAt (timestamp, nullable)
  expiryNoticeSentAt (timestamp, nullable)
  uploadedByUserId (uuid, fk → users.id)
  createdAt, updatedAt
  deletedAt (nullable, soft delete için)

lib/db/src/schema/index.ts'den re-export. pnpm db:push ile migration.

2. Status hesaplama yardımcısı (artifacts/api-server/src/lib/universityContractStatus.ts)
function computeStatus(c) {
  if (!c.effectiveDate || !c.expiryDate) return 'no_dates';
  const now = new Date();
  if (c.expiryDate < now) return 'expired';
  const daysLeft = (c.expiryDate - now) / (1000*60*60*24);
  if (daysLeft <= 30) return 'expiring_soon';
  return 'active';
}

3. Backend API (artifacts/api-server/src/routes/universityContracts.ts)
Auth + university_contracts.view veya .manage permission gerekli. OpenAPI ile dokümante (Orval için).

GET /api/university-contracts — list. Query params: country, year, universityId, status (active/expiring_soon/expired/no_dates), search, page, pageSize
GET /api/university-contracts/:id
POST /api/university-contracts — multipart/form-data (dosya + meta) VEYA önce Object Storage'a presigned upload, sonra metadata kaydet (mevcut objectStorage.ts pattern'ine bak). Tercih: presigned URL pattern (büyük dosya destekli)
PATCH /api/university-contracts/:id — meta + opsiyonel yeni dosya
DELETE /api/university-contracts/:id — soft delete (deletedAt set)
GET /api/university-contracts/:id/download — Object Storage'dan stream et (signed URL'ye redirect veya proxy)
4. Bildirim job'u (artifacts/api-server/src/jobs/universityContractChecker.ts)
Mevcut contractChecker.ts'i (acente sözleşmelerinin expiry kontrolcüsü) birebir referans al — aynı paterni uygula.

Günde bir kez çalışır (mevcut scheduler içine eklenmeli — contractChecker.ts nasıl register ediliyorsa o şekilde)
Tüm universityContracts (deletedAt IS NULL) tara
Her biri için günlerKaldı = expiryDate - today
günlerKaldı = 30 ve lastWarning30SentAt null → uyarı gönder + flag set
günlerKaldı = 14 ve lastWarning14SentAt null → ...
günlerKaldı = 7 ve lastWarning7SentAt null → ...
günlerKaldı = 1 ve lastWarning1SentAt null → ...
günlerKaldı < 0 ve expiryNoticeSentAt null → "Contract expired" gönder + flag set
Hedef alıcılar: tüm aktif admin user'lar + (varsa) üniversiteye atanmış staff
Gönderim: emailQueueTable'a satır ekle (mevcut email queue worker push eder) + notificationsTable'a in-app notification ekle (mevcut notificationDispatcher.ts kullan)
Email konusu: [FindAndStudy] University contract expiring in {N} days: {university.name}
Email body: üniversite adı, destinasyon, tarih, "View" linki (admin paneline derin link: /admin/university-contracts/:id)
Job log'larını mevcut log altyapısıyla yaz (contractChecker.ts nasıl log yazıyorsa).

Üst: filter bar (Country select, Year select, Status select, search input)

Expiry Date: date picker

Notes: textarea
Submit / Cancel
Delete: confirm dialog ("This contract will be moved to trash")
