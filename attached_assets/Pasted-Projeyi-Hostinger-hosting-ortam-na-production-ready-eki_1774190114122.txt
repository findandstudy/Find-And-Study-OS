Projeyi Hostinger hosting ortamına production-ready şekilde deploy edilecek hale getir ve tüm sistemi PageSpeed / Lighthouse açısından en üst seviyede optimize et.

HEDEF:
- Proje Hostinger üzerinde sorunsuz çalışsın
- Production deploy için tam hazır olsun
- pagespeed.web.dev / Lighthouse testlerinde özellikle mobile tarafında mümkün olan en yüksek skorları alsın
- Hedef: Performance, Accessibility, Best Practices ve SEO kategorilerinde 95-100 bandı
- Mümkün olan yerlerde 100/100 hedeflenmeli
- Ancak bunu sadece skor odaklı değil, gerçek kullanıcı deneyimi odaklı yap
- Özellikle Core Web Vitals metriklerini optimize et: LCP, CLS, INP

ÖNEMLİ:
Önce projeyi analiz et ve buna göre doğru deployment modelini seç:
1. Eğer proje static export edilebiliyorsa:
   - build al
   - optimize edilmiş çıktıyı Hostinger public_html yapısına uygun hale getir
   - gerekli rewrite / cache / compression ayarlarını .htaccess ile hazırla
2. Eğer proje Node.js / Next.js SSR / backend gerektiriyorsa:
   - Hostinger web app hosting veya VPS’e uygun production setup hazırla
   - gerekli env yapısını düzenle
   - process management için uygun yapı öner ve uygula
   - reverse proxy, port, build/start komutları, static asset serving yapısını düzgün kur
3. Eğer hibrit yapı varsa:
   - frontend / backend deployment mantığını ayır
   - en performanslı ve en düşük maliyetli Hostinger-compatible çözümü uygula

GENEL GÖREV:
Projeyi baştan sona performans, erişilebilirlik, güvenlik, SEO ve production deploy açısından elden geçir.
Gereksiz tüm yükleri kaldır.
Kod temiz, modüler, maintainable ve production-grade olsun.

PERFORMANCE OPTIMIZATION GÖREVLERİ:
1. Tüm sayfalarda render-blocking kaynakları azalt
2. Gereksiz JavaScript’i temizle
3. Büyük bundle’ları küçült
4. Code splitting uygula
5. Dynamic import kullanılması gereken yerleri belirle
6. Above-the-fold olmayan componentleri lazy load et
7. Tree shaking çalışacak şekilde import yapısını düzelt
8. Unused CSS / JS temizliği yap
9. Kritik CSS’i önceliklendir
10. Font loading stratejisini düzelt:
   - local/self-host mümkünse uygula
   - preload yalnızca kritik fontlarda kullan
   - font-display: swap uygula
   - gereksiz font weight ve subsetleri kaldır
11. Görselleri optimize et:
   - mümkünse WebP / AVIF kullan
   - responsive image yapısı kur
   - width/height değerlerini sabitle
   - CLS oluşmasını engelle
   - hero görseli varsa preload et
   - below-the-fold görselleri lazy load et
12. Video / iframe / harita / embed gibi ağır içerikleri tıkla-yükle veya lazy stratejiyle yönet
13. Third-party scriptleri minimize et
14. Gereksiz analytics/chat/widget/script yüklerini kaldır veya defer et
15. Main thread blocking süresini düşür
16. INP iyileştirmesi için ağır interaction logic’leri optimize et
17. Layout shift oluşturan tüm alanları düzelt
18. Skeleton / placeholder / reserved space kullan
19. Server response ve TTFB tarafında iyileştirme yap
20. Static asset’ler için güçlü cache policy kur
