Add a Popup Ad system to this project. Here is the complete spec:

## Database Schema (shared/schema.ts)
Add two tables using Drizzle ORM + PostgreSQL:

Table 1: `popups`
- id: varchar primaryKey default gen_random_uuid()
- title: text notNull
- content: text notNull
- imageUrl: text (nullable)
- linkUrl: text (nullable)
- linkText: text (nullable)
- targetAudience: text notNull default 'all'  -- values: 'all' | 'agents' | 'specific'
- targetAgencyIds: text[].array() (nullable) -- used when targetAudience='specific'
- status: text notNull default 'draft'  -- values: 'draft' | 'active' | 'archived'
- startsAt: timestamp (nullable)
- expiresAt: timestamp (nullable)
- frequency: text notNull default 'every_session'  -- values: 'every_session' | 'every_login' | 'once_per_user'
- createdBy: varchar notNull
- createdAt: timestamp notNull defaultNow()
- updatedAt: timestamp notNull defaultNow()

Table 2: `popup_dismissals`
- id: varchar primaryKey default gen_random_uuid()
- popupId: varchar notNull
- userId: varchar notNull
- dismissedAt: timestamp notNull defaultNow()
- dontShowAgain: boolean notNull default false
- unique constraint on (popupId, userId)

Also add insertPopupSchema (omit id, createdAt, updatedAt) and insertPopupDismissalSchema.

## Backend Storage (server/storage.ts)
Add to IStorage interface and DatabaseStorage class:
- getPopups(): returns all popups ordered by createdAt desc (join users for creatorName if possible)
- getPopupById(id): returns single popup or undefined
- createPopup(data): inserts and returns the new popup
- updatePopup(id, updates): updates and returns the popup
- deletePopup(id): deletes the popup AND all its popup_dismissals rows first
- getActivePopupsForUser(userId, agencyId, role): returns active popups that:
  - have status = 'active'
  - startsAt is null or in the past
  - expiresAt is null or in the future
  - targetAudience filtering: 'all' = everyone, 'agents' = role must be 'agent', 'specific' = agencyId must be in targetAgencyIds array
- upsertPopupDismissal(data): insert or update on conflict (popupId, userId)
- getUserDismissals(userId): returns all dismissal rows for a user

## API Routes (server/routes.ts)
All admin routes require admin role. All user routes require authentication.

GET    /api/admin/popups           → { success, popups[] }
POST   /api/admin/popups           → body: { title, content, imageUrl?, linkUrl?, linkText?, targetAudience, targetAgencyIds?, status, startsAt?, expiresAt?, frequency } → { success, popup }
PUT    /api/admin/popups/:id       → same body fields (partial) → { success, popup }
DELETE /api/admin/popups/:id       → { success }

GET    /api/popups/active          → returns visible popups for the authenticated user:
  - call getActivePopupsForUser with user.id, user.agencyId, user.role
  - call getUserDismissals for the user
  - filter out: once_per_user if any dismissal exists, any if dontShowAgain=true
  - return { success, popups[] }

POST   /api/popups/:id/dismiss     → body: { dontShowAgain: boolean }
  - call upsertPopupDismissal({ popupId: id, userId: user.id, dontShowAgain })
  - return { success }

## Admin UI Page (client/src/pages/admin/Popups.tsx)
Create a full CRUD admin page with:
- Page title "Pop-up Ads" with a "New Popup" button
- A table listing all popups with columns: Title/Content preview, Target Audience (badge), Frequency, Status (badge: active=default, draft=secondary, archived=outline), Date Range (startsAt → expiresAt), Created At, Actions (edit icon button + delete with confirm dialog)
- Create dialog (opened by "New Popup" button) with a form containing:
  - Title input (required)
  - Content textarea (required)
  - Image URL input + optional file upload button (POST to /api/uploads/content as multipart)
  - Image preview when URL is set
  - Link URL input (optional)
  - Link Text input (optional)
  - Target Audience select: All / All Agents / Specific Agencies
  - If "Specific Agencies" selected: show a scrollable checkbox list of agencies (from GET /api/admin/agencies)
  - Frequency select: Every Session / Every Login / Once Per User
  - Status select: Draft / Active / Archived
  - Start datetime-local input (optional)
  - End datetime-local input (optional)
- Edit dialog (same form, pre-filled)
- All mutations use TanStack Query with cache invalidation on /api/admin/popups

## Popup Renderer Component (client/src/components/PopupRenderer.tsx)
Create a component that:
- Uses useAuthStore to get current user and role
- Uses useLocation (wouter) to detect current path
- Only activates for authenticated agents on the main agent dashboard path (/agent/dashboard or equivalent)
- Fetches /api/popups/active using TanStack Query (only when on the right page)
- Uses sessionStorage to track which popups have been seen this session (key: 'popups_seen_session') and this login (key: 'popups_seen_login')
- Picks the first popup the user hasn't seen yet (checking frequency rules locally)
- Shows it in a shadcn Dialog with:
  - Title (DialogTitle)
  - Optional image (rounded, max-h-64)
  - Content text (whitespace-pre-wrap)
  - Optional CTA link button with ExternalLink icon
  - Bottom row: "Don't show again" checkbox (hidden for once_per_user) + Close button
- On close:
  - Updates sessionStorage/loginStorage based on frequency
  - POSTs to /api/popups/:id/dismiss with { dontShowAgain } 
  - Invalidates /api/popups/active query
- After dismissal, the memo recalculates and shows the next popup if any
